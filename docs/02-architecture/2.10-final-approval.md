# 2.10 Final Architecture Approval

## Decision

Phase 2 is approved for implementation.

## Approved foundations

- full-screen custom view as canonical runtime
- optional supporting Home Assistant integration
- persistent single WebGL stage
- capability-driven device model
- automatic zero-configuration discovery
- semantic command bus and optimistic reconciliation
- versioned canonical data model
- model registry and asset pipeline boundary
- portable preset/blueprint format
- HACS distribution strategy
- Home Configurator Design System and Motion Language

## Required implementation subsystems

- typed event bus
- framework-independent state stores
- command queue
- interaction engine
- transition director
- performance governor
- diagnostics and developer mode
- model scene graph
- plugin lifecycle
- asset optimization pipeline

## Architecture gate

Phase 3 may design concrete UX components, but must not redefine these boundaries. Any necessary foundational change requires a new Architecture Decision Record.

## Risk assessment

Overall risk is moderate and acceptable. The main risks are WebGL performance inside Home Assistant WebViews, model-library scale, editor complexity and cross-version compatibility. Each has a defined boundary and fallback strategy.
