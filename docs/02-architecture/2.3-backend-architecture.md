# 2.3 Backend / Integration Architecture

## Role

The Home Assistant integration is optional. It adds persistent configuration, optimized discovery, model/preset metadata, migrations, diagnostics and authenticated WebSocket APIs.

It does not render UI, duplicate HA entity state or become required for normal device control.

## Integration identity

```text
Domain: home_configurator
Config entries: one per HA installation
Configuration: UI config flow
Platforms: none required in v1
```

## Storage

Versioned repositories manage:

- installation configuration
- presets
- models
- backend settings

Every saved document has a revision number to prevent silent overwrites between multiple editor sessions.

## WebSocket namespaces

```text
home_configurator/system/info
home_configurator/config/*
home_configurator/presets/*
home_configurator/models/*
home_configurator/discovery/*
home_configurator/diagnostics/*
```

Read operations require authentication. Persistent writes require administrator permission in v1.

## Security

- No arbitrary scripts or HTML in model metadata.
- External model URLs are disabled by default.
- File formats, sizes and origins are validated.
- Diagnostics redact tokens, credentials and sensitive paths.
- Missing entities are preserved for recovery rather than deleted.

## Failure behavior

If the integration is absent or incompatible, the frontend falls back to frontend-only mode. Invalid models fall back to generic models. Invalid stored config opens a recovery/validation state rather than a blank dashboard.
