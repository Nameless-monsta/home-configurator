# 2.5 Entity Abstraction Layer

## Goal

Generate a functional dashboard automatically from Home Assistant registries while preserving user control.

## Pipeline

```text
Registry collection
→ normalization
→ area resolution
→ device grouping
→ entity filtering
→ primary entity selection
→ capability detection
→ profile matching
→ model recommendation
→ generated document
→ override reconciliation
```

## Area resolution

```text
1. Explicit override
2. Entity area
3. Parent-device area
4. Sibling-entity area
5. Configured relationship
6. Suggested area
7. Unassigned inbox
```

Suggestions are never silently applied when confidence is low.

## Grouping

Home Assistant device registry ID is the primary grouping key. Entity-only integrations may produce conservative synthetic groups with stable IDs.

Diagnostic, configuration, hidden and disabled entities remain available to the editor but are excluded from primary runtime controls by default.

## Automatic inference

Each discovered visual device receives:

- room placement
- primary entity
- supporting semantic roles
- capability candidates
- profile match
- model recommendation
- confidence and explanation

## Reconciliation

Registry refreshes preserve user names, ordering, room overrides, model selection and controls. New devices enter an admission workflow. Missing devices remain recoverable and are not automatically deleted.
