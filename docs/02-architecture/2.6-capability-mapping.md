# 2.6 Device Capability Mapping

## Goal

Translate inconsistent Home Assistant domains, attributes and supported-feature flags into stable interaction capabilities.

## Pipeline

```text
Raw entity metadata
→ capability detector
→ normalized capability
→ interaction resolver
→ semantic command
→ renderer feedback
```

## Core capability families

- power
- brightness
- color and color temperature
- climate temperature, HVAC, fan and swing
- cover position and tilt
- media playback, seek, volume and source
- vacuum start, pause, stop, return, locate and fan speed
- battery and availability
- scene activation
- sensor and binary-state presentation
- generic action triggers

## Capability contract

Every capability includes:

- stable ID
- source entity role
- supported values/range/options
- evidence and confidence
- state normalization
- semantic commands
- preferred controls
- accessible fallback
- optimistic-preview policy

## Control resolution

The renderer asks which capabilities are available. A control registry selects the appropriate interaction, such as:

- power → toggle
- brightness → continuous gesture/slider
- color → 3D color sphere
- target temperature → temperature dial
- cover position → position gesture
- media playback → transport rail
- vacuum actions → combined vacuum control experience

Capabilities are manufacturer-independent and extensible through namespaced IDs.
