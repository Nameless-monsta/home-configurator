# 2.7 Model Registry

## Goal

Resolve each visual device to a validated 3D representation without hardcoding assets into device controllers.

## Matching priority

```text
1. Explicit user override
2. Exact manufacturer/model alias
3. Manufacturer family
4. Device profile default
5. Capability/category fallback
6. Abstract fallback object
```

## Model metadata

Each entry defines:

- stable model ID and version
- source and format
- display name and thumbnail
- category/profile aliases
- license and attribution
- bounding box, pivot and visual center
- camera/framing presets
- material bindings
- animated parts
- exploded-view metadata
- LOD variants
- capability/state mappings
- performance estimates

## Asset formats

GLB is the canonical model format. Compressed textures and optional Draco/KTX2 assets may be supported by the asset pipeline.

## Runtime behavior

The registry resolves metadata before loading assets. The renderer selects LOD based on performance tier and framing. Failed or missing models fall back without breaking navigation or controls.

## Distribution

V1 includes a small generic core library. Manufacturer-specific or large content may later be distributed as separate model packs. User models remain local and require validation.
