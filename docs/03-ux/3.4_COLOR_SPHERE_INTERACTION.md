# 3.4 Color Sphere Interaction

**Status:** Complete  
**Phase:** 3 — UX and Visual Design

## Objective

Define a premium 3D light-color control that replaces the conventional flat hue wheel.

The sphere must support:

- hue and saturation selection by rotation;
- immediate visual preview on the active 3D light;
- two-finger brightness control without resizing the sphere;
- accessible non-gesture alternatives;
- responsive use on iPhone, iPad and desktop.

## Visual concept

The color selector is a physically rendered sphere inside the same scene language as the active device.

It is not:

- a decorative orb in the background;
- a generic rainbow wheel pasted over the interface;
- a second hero object competing with the selected device.

It appears only inside the light-control state and uses controlled scale, lighting and spacing.

## Color mapping

The sphere surface maps:

```text
Longitude → hue, 0–360°
Latitude/equatorial distance → saturation
Neutral/polar zone → reduced saturation
```

Brightness is not encoded by sphere size. Sphere geometry remains stable.

## Primary gestures

### One-finger rotation

- drag rotates the sphere;
- selected surface point remains under or near the gesture;
- hue and saturation update continuously;
- inertia is restrained and stops quickly enough for precision;
- the active lamp previews the selected color immediately.

### Two-finger vertical luminosity gesture

```text
Two fingers move upward → increase brightness
Two fingers move downward → decrease brightness
```

Rules:

- sphere size remains unchanged;
- emissive intensity, local light and readout respond continuously;
- gesture starts from current brightness;
- pointer count change cancels or safely re-arbitrates the session;
- accidental one-finger rotation is suppressed while brightness is active.

### Tap

Tap selects the exact visible sphere point without requiring drag.

## Optional desktop input

- drag rotates sphere;
- wheel or vertical slider adjusts brightness only when the color control is focused;
- keyboard arrows adjust hue/saturation;
- Page Up/Down or dedicated controls adjust brightness.

## Readouts

Visible readouts:

- human-friendly color description where available;
- hue value;
- saturation percentage;
- brightness percentage;
- optional Kelvin value when color-temperature mode is active.

Readouts remain secondary to the sphere and device preview.

## Color and brightness modes

```text
Color
White temperature
Presets/effects
```

The sphere is shown only for color-capable lights.

For color-temperature-only lights, use the climate-like thermal control rather than simulating unsupported RGB.

## Preview and command behavior

```text
Gesture movement
→ renderer preview at frame rate
→ readout update at throttled rate
→ HA command throttled/coalesced
→ final command on release
→ confirmation reconciliation
```

Recommended service throttle:

```text
80–150 ms during gesture
Immediate final commit on release
```

## Lighting response

Brightness modifies:

- device emissive intensity;
- plausible local illumination;
- subtle bloom;
- nearby reflection contribution;
- numeric readout.

Color modifies:

- emissive color;
- local light color;
- controlled environmental reflection;
- small semantic color indicator.

UI typography and navigation remain monochrome.

## Sphere states

- `idle`
- `hover/focus`
- `rotating`
- `brightness_adjusting`
- `pending`
- `confirmed`
- `disabled`
- `unavailable`

## Motion

- sphere enters after the light object settles;
- rotation follows direct manipulation with minimal lag;
- selection marker uses a subtle spring;
- brightness preview uses smooth continuous interpolation;
- on exit, the sphere de-emphasizes before control removal;
- reduced-motion mode removes idle drift and inertial flourish.

## Responsive behavior

### iPad landscape

- sphere appears in the contextual control region;
- minimum useful rendered diameter: approximately 180–240 px;
- device and sphere remain visible together;
- brightness readout stays adjacent.

### iPhone landscape

- sphere may become the main contextual control;
- device preview remains partially visible;
- nonessential metadata hides.

### iPhone portrait

- sphere appears above the bottom dock;
- brightness fallback slider remains reachable;
- no horizontal overflow.

### Desktop

- sphere supports pointer hover preview only as enhancement;
- precise keyboard operation is available.

## Accessibility fallback

Provide:

- hue slider;
- saturation slider;
- brightness slider;
- preset color swatches;
- numeric announcements;
- clear labels and focus states.

The sphere is never the only method of choosing color.

## Error behavior

When a light rejects a selected color:

- retain the user's preview briefly;
- display a restrained error;
- animate back to confirmed state;
- preserve brightness if only color failed.

When the entity becomes unavailable, the sphere remains visible but disabled and desaturated.

## Acceptance criteria

- one-finger rotation changes hue and saturation;
- two-finger vertical motion changes brightness;
- the sphere never scales to represent brightness;
- the actual light model previews color and luminosity live;
- commands are throttled and coalesced;
- final release commits the precise value;
- unsupported color modes never appear;
- keyboard and slider fallbacks provide full control;
- UI chrome remains monochrome while device-generated color remains local.
