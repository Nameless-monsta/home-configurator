# 3.6 — Media Interaction

**Status:** Complete  
**Depends on:** 3.1 Navigation Shell, 3.2 Room Navigation, 3.3 Device Configurator

## Objective

Define one coherent cinematic interaction model for televisions, Apple TV, speakers, receivers, and grouped media systems without reducing them to conventional dashboard cards.

## Experience principles

- The active media device remains the visual centre.
- Power, source, playback, and volume are expressed as states of one object.
- Controls appear contextually and disappear when no longer relevant.
- Artwork and device-generated light may influence the stage, but never overpower navigation.
- One visual media system may compose multiple Home Assistant entities.

## Supported capabilities

- Power
- Play, pause, stop
- Previous and next
- Seek
- Volume and mute
- Source selection
- Application selection
- Media metadata
- Artwork
- Channel control where available
- Grouped speakers
- Remote actions
- Availability and connection state

## Core states

```text
OFF
STANDBY
IDLE
BUFFERING
PLAYING
PAUSED
ERROR
UNAVAILABLE
```

## Default composition

### Television

- 3D television or display is the hero object.
- Screen state is represented physically through the display material.
- Metadata aligns beside or beneath the object depending on viewport.
- Transport controls remain secondary to content identity.

### Apple TV or streaming box

- The streaming device may be the hero object when selected directly.
- When composed with a television, Apple TV becomes a source layer within one visual system.
- Selecting a source must not create a separate card or duplicate volume controls.

### Speaker

- Speaker model remains central.
- Playback state may drive restrained vibration, diaphragm movement, or low-opacity waveform response.
- Multi-room groups are shown as a coordinated set, not a list of entities.

## Primary interaction model

### Tap

- Select device.
- Reveal primary controls.
- Toggle play/pause only when the current interaction surface explicitly indicates this action.

### Horizontal drag

- Seek through media when seek capability exists.
- Not enabled globally to avoid conflict with room navigation.

### Vertical drag

- Adjust volume when the volume interaction mode is active.
- Up increases volume, down decreases volume.

### Two-finger vertical gesture

- Optional precision volume control.
- Used when one-finger drag is reserved for object rotation.

### Long press

- Open contextual actions such as source, application, output group, or remote commands.

## Volume interaction

- Volume is normalized from `0..1`.
- Visual feedback is immediate and optimistic.
- Service calls are throttled.
- Device confirmation reconciles the preview smoothly.
- The user sees a numeric or percentage readout only during adjustment.
- Mute is a discrete action and never inferred from zero volume alone.

## Playback controls

Primary transport set:

```text
Previous · Play/Pause · Next
```

Optional actions:

```text
Stop · Seek · Shuffle · Repeat
```

Rules:

- Unsupported controls are omitted.
- The play/pause control morphs rather than swapping abruptly.
- Pending actions show restrained state feedback.
- Repeated taps are debounced.

## Source and application selection

- Source selection opens a focused contextual rail or overlay.
- The current source remains clearly marked.
- Sources are ordered by user preference, then device order.
- Long source lists support search or grouped categories.
- The 3D object remains visible behind the selector.
- Selecting a source closes the selector only after acknowledgement.

## Metadata presentation

Possible metadata:

- Title
- Artist
- Album
- Series and episode
- Source/application
- Playback progress
- Duration
- Artwork

Priority order:

```text
Title
Source or artist
Progress
Secondary metadata
```

Missing metadata must not leave empty labels.

## Artwork rules

- Artwork is optional.
- It may tint local screen emission or a restrained reflection field.
- It must not recolour the permanent shell.
- Low-resolution artwork is softened or contained rather than enlarged aggressively.
- Sensitive media information may be hidden through a privacy setting.

## Composed media systems

Example:

```text
Television
+ Apple TV
+ AV receiver
+ ambient light
= one visual media system
```

The system resolves:

- one power state;
- one active source;
- one volume owner;
- one playback owner;
- one presentation model.

Conflicts are surfaced in the editor, not during normal use.

## Remote actions

Remote actions may include:

- directional navigation;
- select/back/home;
- menu;
- channel controls;
- application launch.

These open in a dedicated control mode and do not appear permanently.

## Grouped audio

- Group membership is visible contextually.
- Group volume and device volume are distinct.
- Removing a speaker from a group requires an explicit action.
- Unavailable speakers remain visible but de-emphasized.

## Motion and lighting

### Playing

- Screen or speaker emission activates.
- Very subtle ambient motion may resume.

### Paused

- Motion settles.
- Screen remains active but reduced in energy.

### Buffering

- Use a restrained cyclic material or progress response.
- Avoid generic loading spinners over the object.

### Off

- Emissive surfaces fade down.
- Camera may return to a neutral hero pose.

## Responsive behavior

### iPad landscape

- Hero object occupies the centre-right or centre-left depending on controls.
- Metadata and transport controls coexist without scrolling.
- Source selector appears as a contextual side rail.

### iPhone landscape

- Transport controls form a compact bottom row.
- Metadata is reduced to title, source, and progress.
- Source selection becomes a larger overlay.

### iPhone portrait

- Object remains in the upper portion.
- Transport controls occupy the bottom dock.
- Advanced actions are behind one contextual menu.

### Desktop

- Hover may preview source or remote actions.
- Keyboard shortcuts may control playback when focus is inside the media experience.

## Accessibility

- All transport actions have accessible names.
- Volume exposes slider semantics.
- Playback changes are announced.
- Artwork has alternative text only where meaningful.
- Gesture actions have visible button alternatives.
- Keyboard controls must not intercept unrelated browser or Home Assistant shortcuts.

## Error and unavailable behavior

- Service failure restores confirmed state smoothly.
- Unsupported source changes show a concise error.
- Unavailable devices preserve their model and metadata where safe.
- A failure in one composed entity must not crash the whole media system.

## Acceptance criteria

1. Television, Apple TV, speakers, and receivers can share one interaction language.
2. Multi-entity systems do not produce duplicate controls.
3. Volume adjustment is immediate, throttled, and reconciled.
4. Source selection remains contextual and reversible.
5. Unsupported capabilities are omitted automatically.
6. Artwork influences only local presentation.
7. All gestures have accessible alternatives.
8. Media state changes affect the 3D object and lighting.
9. iPad landscape requires no page scrolling for primary controls.
10. Errors remain local to the affected action or entity.
