# 4.2 вЂ” Runtime Core

**Status:** Complete  
**Milestone version:** `0.2.0`

## Delivered

The runtime core is now implemented as the `@home-configurator/runtime` package and exercised by the runnable `@home-configurator/dashboard` application.

The milestone includes:

- deterministic application bootstrap and shutdown;
- typed dependency injection and service registry;
- typed, re-entrancy-safe event bus;
- centralized frame scheduler with ordered tasks;
- diagnostics entries, counters, gauges and subscriptions;
- validated runtime configuration loading;
- plugin registration, startup, isolation and disposal;
- asset registration, caching and failure reporting;
- Home Assistant adapter port and mock adapter;
- runtime lifecycle events and service tokens;
- accessible runtime-status shell for development;
- automated unit tests and production builds.

- current runtime phase;
- scheduler frame count;
- event count;
- safe-area-aware iPhone/iPad shell;
- graceful shutdown on page exit.

## Acceptance criteria

- [x] Runtime can boot and stop deterministically.
- [x] Dependencies resolve through typed service tokens.
- [x] Circular dependencies are rejected.
- [x] Events retain deterministic order during re-entrant emission.
- [x] Scheduler tasks execute in stable priority order.
- [x] Diagnostics expose frame and lifecycle state.
- [x] Plugin and asset failures are isolated and reported.
- [x] A mock Home Assistant boundary is available without coupling core to WebSocket implementation.
- [x] Runtime package builds to ESM with declarations and source maps.
