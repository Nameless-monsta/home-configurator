# 4.3 — Graphics Engine

**Status:** Complete and CI validated  
**Milestone version:** 0.3.0

## Delivered

- one persistent Three.js `WebGLRenderer` bound to the dashboard canvas;
- runtime-scheduler bridge with a single centralized render task;
- responsive viewport controller with DPR limits and visual-viewport handling;
- perspective camera rig with deterministic pose transitions, reduced-motion support and object framing;
- ID-addressable scene graph with explicit parent ownership;
- restrained warm studio-lighting rig with quality-aware shadow configuration;
- ACES filmic tone mapping, sRGB output and color-managed post-processing output;
- optional bloom pipeline with essential, balanced and full quality profiles;
- quality-controlled shadows, pixel ratio and LOD policy;
- GLTF/GLB loading with caching, preloading, eviction, validation and safe cloned instances;
- HDR environment loading through RGBE and PMREM processing;
- WebGL context-loss detection and restoration handling;
- centralized GPU resource tracking and disposal;
- frame, viewport, DPR, draw-call, triangle, geometry, texture, shader-program, quality and context diagnostics;
- generic original 3D lamp fallback used by the milestone demo;
- safe-area-aware iPhone and iPad dashboard stage;
- unit coverage for scene ownership, camera transitions, framing, quality profiles, resource disposal and context recovery.

## Package boundary

The implementation lives in `packages/graphics` and depends only on the Runtime Core contract and Three.js. The dashboard owns DOM composition and mounting. The graphics package owns rendering, scene hierarchy, camera, lighting, quality, environments, post-processing, models and GPU-resource disposal.

## Runtime flow

```text
Dashboard canvas
→ attachGraphicsRuntime
→ GraphicsEngine
→ SceneGraph + CameraRig + Lighting + Environment
→ LOD + PostProcessing + ResourceTracker
→ RuntimeScheduler graphics.render task
→ WebGLRenderer
→ diagnostics
```

## Current demo

The dashboard mounts one persistent WebGL canvas and an original procedural lamp assembled from physical materials, an emissive bulb and local light. The object floats subtly while the stage reports live frame, draw, triangle, texture and quality diagnostics.

The procedural lamp is a generic fallback fixture, not a final manufacturer model. Real models are resolved later through the approved model registry and asset pipeline.
