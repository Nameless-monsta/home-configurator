# 4.6.3 — Configurator Framework

**Status:** Implemented, pending CI validation  
**Milestone version:** 0.6.3

## Delivered

- generic configurator document, section, field, action and validation contracts;
- adapter boundary for committing changes and invoking actions;
- immutable snapshot subscriptions;
- pending-value and dirty-state tracking;
- undo and redo history;
- reset and apply lifecycle;
- validation issues and save blocking;
- confirmed-value merge after successful commits;
- explicit null-value preservation;
- collapsible section renderer;
- loading, empty, unavailable and validation states;
- generic toggle, range, number, select, text and status hosts;
- responsive configurator presentation;
- dashboard mapping from selected canonical Home Assistant devices;
- unit tests for state, history, validation, save, action dispatch and guarded fields.

## Scope boundary

This milestone provides the generic host framework. The polished reusable control library belongs to 4.6.4, and capability-specific device panels belong to 4.6.5.

## Validation gate

The milestone is complete only after GitHub Actions passes formatting, ESLint, strict TypeScript, tests and production builds.
