# 4.6.4 — Control Library

**Status:** Complete and CI validated  
**Milestone version:** 0.6.4

## Delivered

- reusable control registry with replaceable renderers;
- toggle control with switch semantics;
- slider with live value output and unit support;
- numeric input with minimum, maximum and step constraints;
- select control;
- segmented selector with radio semantics;
- text input;
- colour input;
- read-only status output;
- centralised value parsing;
- disabled, read-only and saving-state propagation;
- HTML escaping for labels, options and values;
- responsive control styling;
- keyboard and focus-visible support through native controls and semantic buttons;
- unit tests for rendering, parsing, escaping, disabled state and renderer replacement.

## Architecture

`ControlLibrary` owns the mapping between `ConfiguratorFieldKind` and `ControlRenderer`. `UiConfigurator` delegates rendering and value parsing to that registry, allowing future plugins or device panels to replace a renderer without changing the configurator framework.

## Scope boundary

Capability-specific Home Assistant panels, final iyO motion choreography and full responsive validation were completed in 4.6.5 through 4.6.8.

GitHub Actions passed Prettier formatting, ESLint, strict TypeScript, unit tests and production builds.
