# 4.6.4 Control Library

## Status

Implementation complete. CI validation is running on pull request #12 before merge.

## Delivered

- reusable control registry with replaceable renderers
- toggle control with switch semantics
- slider with live value output and unit support
- numeric input with minimum, maximum and step constraints
- select control
- segmented selector with radio semantics
- text input
- colour input
- read-only status output
- centralised value parsing
- disabled, read-only and saving-state propagation
- HTML escaping for labels, options and values
- responsive control styling
- keyboard and focus-visible support through native controls and semantic buttons
- unit tests for rendering, parsing, escaping, disabled state and renderer replacement

## Architecture

`ControlLibrary` owns the mapping between `ConfiguratorFieldKind` and `ControlRenderer`. `UiConfigurator` delegates both rendering and value parsing to that registry, allowing future plugins or device panels to replace a renderer without changing the configurator framework.

## Out of scope

- capability-specific Home Assistant device panels
- final iyO motion choreography
- final cross-device responsive validation

These remain scheduled for 4.6.5, 4.6.6 and 4.6.7.
