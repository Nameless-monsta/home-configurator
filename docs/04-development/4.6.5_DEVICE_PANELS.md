# 4.6.5 — Device Panels

**Status:** Complete and CI validated  
**Milestone version:** 0.6.5

## Delivered

- generic `DevicePanelSource` contract independent of Home Assistant internals;
- priority-ordered, replaceable device-panel registry;
- built-in light, climate, cover, media, sensor and fallback providers;
- multi-panel composition for devices exposing several capability families;
- identity metadata section;
- Home Assistant adapter for canonical devices and entity states;
- demo devices for light, climate, media, cover and sensors;
- provider tests covering resolution, output and replacement.

## Architecture

Home Assistant canonical devices are translated by the dashboard adapter into a generic panel source. The UI registry resolves one or more providers and produces a standard configurator document. `UiConfigurator` and the control library remain unaware of Home Assistant domains.

## Scope boundary

The panels defined the capability-driven UI surface. Live service-command dispatch and state reconciliation belong to 4.7, while vacuum, lock and other specialist panels remain future extensions.

## Validation

GitHub Actions passed Prettier formatting, ESLint, strict TypeScript, unit tests and production builds.
