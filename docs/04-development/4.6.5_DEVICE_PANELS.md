# 4.6.5 Device Panels

## Status

Implementation complete on the feature branch. Validation pending.

## Delivered

- Generic `DevicePanelSource` contract independent of Home Assistant internals.
- Priority-ordered, replaceable device panel registry.
- Built-in light, climate, cover, media, sensor and fallback providers.
- Multi-panel composition for devices exposing several capability families.
- Identity metadata section.
- Home Assistant adapter for canonical devices and entity states.
- Demo devices for light, climate, media, cover and sensors.
- Provider tests covering resolution, output and replacement.

## Architecture

Home Assistant canonical devices are translated by the dashboard adapter into a generic panel source. The UI registry resolves one or more providers and produces a standard configurator document. `UiConfigurator` and the control library remain unaware of device domains.

## Out of Scope

- Final service-command dispatch and reconciliation for every control.
- Vacuum, lock and advanced specialist panels.
- Final iyO motion choreography.
- Responsive validation across the full device matrix.
