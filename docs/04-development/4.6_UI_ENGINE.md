# 4.6 — UI Engine

**Status:** Implemented, pending CI validation  
**Milestone version:** 0.6.0

## Delivered

- reusable `@home-configurator/ui` package with no Home Assistant entity hardcoding;
- persistent full-screen application shell around the existing WebGL stage;
- restrained top navigation bar with room and device quick access;
- accessible navigation dropdown with deterministic tabs and selection state;
- room and device context headings integrated with the 3D composition;
- capability-driven configurator rail generated from canonical device discovery;
- semantic device-action events routed through the Home Assistant Engine boundary;
- overlay ownership for navigation, configurator and diagnostics modes;
- Escape-key dismissal and keyboard-focus treatment;
- iPad, iPhone and desktop responsive layouts;
- safe-area support for full-screen iOS use;
- reduced-motion behavior;
- runtime, Home Assistant and interaction diagnostics HUD;
- pure UI state reducer and portable navigation-model tests;
- dashboard demonstration using the existing automatic Home Assistant fixture.

## Composition flow

```text
Home Assistant confirmed snapshot
→ UI state model
→ room/device navigation
→ selected context
→ capability-driven configurator rail
→ semantic device action
→ Home Assistant Engine command boundary
```

The UI Engine owns presentation, overlay state and accessible DOM controls. It does not own Home Assistant transport, command translation, renderer state, gesture recognition or camera implementation.

## Design constraints

The implementation follows the approved visual direction:

- warm dark monochrome background;
- one dominant 3D hero object;
- minimal typography and hairline separators;
- no green accents or decorative background networks;
- compact navigation that opens downward;
- narrow device-control rail on larger displays and a bottom sheet on phones;
- interface and 3D stage presented as one composition.

## Validation gate

The milestone is complete only when GitHub Actions passes:

- formatting;
- ESLint;
- strict TypeScript;
- UI and existing package tests;
- all production builds.
