# 4.7.2.1 — Runtime Device Store

**Status:** Implementation in progress on `feature/runtime-state-v0.7.2`  
**Milestone version:** 0.7.2

## Scope

Create the application-owned runtime state boundary that sits between external state producers and UI or graphics consumers. This package is intentionally independent from Home Assistant, the UI package and the Graphics package.

## Delivered in this slice

- standalone `@home-configurator/runtime-state` package;
- canonical `DeviceRuntime` with confirmed, optimistic and effective state;
- command-aware confirmation and rollback behaviour;
- availability, connectivity and selection state;
- bounded transition history through a reusable ring buffer;
- latency, confirmation, rollback and availability metrics;
- `DeviceRegistry` indexes by device, entity and room;
- `DeviceStore` with device-level, room-level and structural subscriptions;
- central `SelectionManager` for room, selected device, hover and focus state;
- aggregate runtime-state diagnostics;
- strict unit and contract tests for reconciliation, indexing, subscriptions, selection and diagnostics.

## Dependency boundary

`@home-configurator/runtime-state` has no dependency on:

- `@home-configurator/home-assistant`;
- `@home-configurator/ui`;
- `@home-configurator/graphics`.

External integrations will translate their own snapshots into this package in later 4.7.2 slices.
