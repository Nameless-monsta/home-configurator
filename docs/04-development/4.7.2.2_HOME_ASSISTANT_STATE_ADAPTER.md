# 4.7.2.2 — Home Assistant State Adapter

**Status:** Complete and CI validated  
**Milestone version:** 0.7.2  
**Pull request:** #20

## Scope

Connect the Home Assistant Engine to the application-owned Runtime device state subsystem without exposing Home Assistant entity details to UI or graphics consumers.

## Delivered

- pure translation from canonical Home Assistant devices and states into Runtime descriptors and confirmed state;
- normalized capability values for power, brightness, colour, colour temperature, climate, media, covers, locks and sensors;
- semantic-command translation into Runtime optimistic patches;
- command dispatch wrapper that applies optimistic state before Home Assistant service calls;
- independent reconciliation of concurrent capability commands;
- confirmation tolerance for brightness, volume, colour, temperature and cover position;
- rollback on service failure, timeout, stale connection, device removal or adapter disposal;
- continuous-command coalescing;
- full-snapshot hydration and removal of disappeared Home Assistant devices;
- dashboard integration and Runtime-state diagnostics;
- unit and contract tests for translation, confirmation, concurrent commands, rollback, timeout, stale state and removal.

## Runtime enhancement

`DeviceRuntime` supports multiple concurrent optimistic command layers. Each command can be confirmed or rolled back independently while unrelated authoritative updates continue to refresh confirmed state.

## Dependency boundary

The adapter belongs to `@home-configurator/home-assistant`. Runtime remains unaware of Home Assistant types, entities, transports and command semantics.
