# 4.7.2.3 — Selected-Device Model Binding

**Status:** Complete and CI validated  
**Milestone version:** 0.7.3  
**Pull request:** #22

## Scope

Bind the selected Runtime device to the graphics model so authoritative Home Assistant state and optimistic command state produce visible feedback through one application-owned state boundary.

## Delivered

- reusable `SelectedDeviceLightBinding` in `@home-configurator/graphics`;
- dashboard navigation synchronised with Runtime device selection;
- selected-device updates driven by `DeviceStore.effectiveState`;
- light power mapped to emissive and point-light output;
- normalised brightness mapped to material and point-light intensity;
- Home Assistant colour state mapped to the model colour;
- unavailable or disconnected devices rendered with reduced opacity and no emitted light;
- pending commands represented with a temporary scale and intensity lift;
- authoritative confirmation, rollback and external Home Assistant updates reflected without direct UI-to-model coupling;
- unit tests for live binding, pending confirmation and unavailable-device rendering.

## State boundary

The Graphics package consumes Runtime snapshots only. It remains unaware of Home Assistant entities, services and transports. The Home Assistant adapter produces state, Runtime owns reconciliation, and the model binding renders the selected device's effective state.
