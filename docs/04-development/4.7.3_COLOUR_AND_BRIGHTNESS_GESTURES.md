# 4.7.3 — Colour and Brightness Gestures

**Status:** Complete and CI validated  
**Milestone version:** 0.7.4  
**Pull request:** #23

## Scope

Add direct manipulation for the selected colour-capable light while preserving the Runtime and Home Assistant state boundaries.

## Delivered

- horizontal drag maps to hue;
- vertical drag maps to saturation;
- two-finger vertical movement maps to brightness;
- mouse wheel maps to brightness;
- intermediate updates are dispatched as continuous semantic commands;
- release emits a final semantic command;
- optimistic Runtime state drives immediate 3D feedback;
- authoritative Home Assistant confirmation or rollback remains responsible for reconciliation;
- deterministic mapping and command-contract tests;
- robust hue wrapping for large positive and negative drag distances.

## Interaction contract

The fallback lamp is now a light-control target rather than an orbit target. Tap still focuses the selected model. Drag controls colour, while two-finger vertical movement and wheel input control brightness.
