# 4.7.4 — Prototype Validation and Device Testing

## Status

Complete and CI validated.

## Scope

This milestone closes the first interactive prototype by validating the live dashboard across accessibility, responsive behaviour, reduced motion, fallback rendering, diagnostics and real-device operation.

## Automated validation

- deterministic phone, tablet portrait, tablet landscape and desktop viewport classification;
- keyboard-accessible interactive stage semantics;
- accessible stage naming for screen readers;
- reduced-motion and coarse-pointer state publication;
- generic fallback-model availability;
- diagnostics availability;
- machine-readable validation result through root data attributes;
- `home-configurator:prototype-validation` browser event for diagnostics and test harnesses.

## Manual acceptance matrix

The prototype acceptance matrix covers:

1. iPad Safari in portrait and landscape;
2. desktop Safari or Chrome with mouse wheel brightness;
3. phone width with no horizontal overflow;
4. keyboard-only navigation and activation;
5. VoiceOver navigation and control naming;
6. reduced-motion mode;
7. unavailable Home Assistant device state;
8. missing or invalid model asset using the generic fallback;
9. real Home Assistant colour-capable light confirmation and rollback;
10. diagnostics visibility for frame, gesture, transition and command state.
