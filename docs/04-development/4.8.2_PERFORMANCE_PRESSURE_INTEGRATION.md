# 4.8.2 — Performance Pressure Integration

## Status

Implementation in progress.

## Scope

This slice connects the renderer-budget and long-task monitors to the live graphics runtime.

## Delivered

- automatic registration with `attachGraphicsRuntime`;
- per-frame renderer resource sampling;
- browser `longtask` observation where supported;
- edge-triggered warning, critical and recovery diagnostics;
- active and critical pressure gauges;
- long-task count, total duration and maximum-duration gauges;
- one-tier quality downgrade for critical resource pressure;
- cooldown protection so resource pressure cannot repeatedly collapse quality;
- ESSENTIAL quality floor shared with frame-time adaptation;
- deterministic unit coverage for resource-pressure decisions.

## Behaviour

Frame-time adaptation remains responsible for sustained timing pressure. Resource-pressure adaptation responds only to newly emitted critical renderer-budget alerts. Both paths use `GraphicsEngine.setQualityTier`, so LOD, pixel ratio, lighting and post-processing remain consistent.

## Validation gate

The slice is complete only after formatting, linting, strict TypeScript, all tests and production builds pass on the final pull-request head.
