# 4.8.3 — Asset Cache and Sustained Profiling

## Status

Implementation in validation.

## Scope

This slice bounds model-cache growth and defines the sustained-device profiling gate required to close 4.8.

## Asset-cache policy

- least-recently-used eviction;
- default maximum of twelve cached model sources;
- cache hits refresh recency;
- in-flight model loads are protected from eviction;
- failed loads are removed from the cache and policy state;
- manual eviction and full clear remain available;
- diagnostics expose entries, maximum entries, hits, misses and evictions;
- eviction events identify the removed model ids.

## Sustained profiling gate

Run the prototype for at least fifteen minutes on each target surface:

1. iPad Safari landscape;
2. iPad Safari portrait;
3. desktop Safari or Chrome;
4. one lower-power mobile or tablet profile.

During each run:

- navigate repeatedly across all demo rooms;
- select and deselect the hero device;
- perform repeated colour and brightness gestures;
- trigger at least twenty Home Assistant commands;
- background and foreground the tab at least three times;
- load more unique model ids than the cache limit;
- confirm cache entries return to the configured maximum;
- confirm no repeated budget-warning spam occurs;
- record FPS, average frame time, P95 frame time, long tasks and active pressure.

## Exit criteria

4.8 may close when:

- formatting, lint, strict TypeScript, tests and builds pass;
- cache growth is bounded under repeated model loading;
- no critical renderer pressure persists after automatic tier reduction;
- command and gesture behaviour remains correct during sustained runs;
- hidden-tab suspension resumes without an animation backlog;
- any device-specific exceptions are documented rather than hidden.
