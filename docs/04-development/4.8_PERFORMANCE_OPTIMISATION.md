# 4.8 — Performance Optimisation

## Status

Implementation in progress.

## Objective

Protect input responsiveness, Home Assistant command reliability and state accuracy by reducing visual quality before interaction quality degrades.

## First slice

- deterministic adaptive quality controller;
- FULL, BALANCED and ESSENTIAL frame-time budgets;
- downgrade after sustained average frame-time overrun;
- downgrade when more than ten percent of frames exceed 1.5 times the active budget;
- longer stable window before upgrading;
- explicit reset when quality is selected manually;
- unit coverage for downgrade, slow-frame and upgrade hysteresis.

## Next integration

1. Feed scheduler frame timing into the controller.
2. Apply tier decisions through `GraphicsEngine.setQualityTier`.
3. Record downgrade reason, average frame time and slow-frame ratio in diagnostics.
4. Suspend nonessential animation and rendering while the document is hidden.
5. Expose rolling FPS, average frame time and P95 frame time.
6. Validate iPad landscape behaviour and repeated gesture interaction.

## Validation gate

The milestone remains open until formatting, linting, strict TypeScript, all tests and production builds pass on the final pull-request head.
