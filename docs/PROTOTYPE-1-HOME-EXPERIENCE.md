# Prototype 1 — Home Experience

## Goal

Turn the Phase 5 shell into a visually reviewable, production-oriented Home screen where the favourite device is the clear hero, rooms feel ambient rather than tabular, and the interface can be judged directly against the IYO-inspired direction.

## Scope

- Home screen visual hierarchy
- Favourite-device hero composition
- Ambient room summary
- Spatial navigation polish
- Responsive desktop and iPad behaviour
- Reduced-motion parity

## Acceptance

- The favourite device reads as the centre of the experience within one second.
- Navigation remains secondary to the object.
- Room access is visible without looking like a conventional dashboard grid.
- Device state remains readable without persistent utility panels.
- Existing Home Assistant command and state paths remain unchanged.
